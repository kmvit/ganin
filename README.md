# sd2-kmv.ru — статика

Одностраничный сайт ЗАО «Стройдеталь-2» (Пятигорск). Чистая статика:
`index.html` + `assets/`. Никакого билд-степа.

## Запуск локально

```bash
python3 -m http.server 8000
# открыть http://localhost:8000
```

## Структура

```
index.html
assets/
├── css/styles.css     — дизайн-система и все секции
├── js/main.js         — скролл-видео + reveal + меню
├── hero-frames/       — 120 кадров WebP для hero scroll-эффекта
└── img/               — фото объектов/продукции (плейсхолдеры)
mp_.mp4                — исходное видео, источник для кадров
```

## Скролл-видео в hero — как пересобрать кадры

Если меняется исходник `mp_.mp4`, кадры пересобираются один раз
локально (нужен `ffmpeg` + `cwebp` — `brew install ffmpeg webp`):

```bash
# 0. удалить старые кадры
rm -f assets/hero-frames/frame-*.webp

# 1. вытащить JPG-кадры на 15 fps и шириной 1280
ffmpeg -y -i <ваше-видео.mp4> -vf "fps=15,scale=1280:-2" -q:v 4 \
       assets/hero-frames/frame-%03d.jpg

# 2. перегнать в WebP (q=55 — баланс веса и качества)
cd assets/hero-frames
for f in *.jpg; do
  cwebp -q 55 -m 6 -mt -quiet "$f" -o "${f%.jpg}.webp"
done
rm *.jpg
```

После пересборки **обязательно** обновить в `assets/js/main.js`
константу `FRAME_COUNT` — должна равняться количеству файлов
в `hero-frames/` (обычно `fps × длительность`).

## Деплой

Залить весь корень проекта на хостинг как есть (`index.html`
должен быть в корне сайта). Файл `mp_.mp4` можно не выкладывать
в прод — он нужен только для пересборки кадров.
